OO v1 (Deprecated) Stored display options both centrally and in each mediator. 312 lines of code. This implemenation had a display_options mediator (as a Subject for a change, usually mediators are observers of model subjects). Then mediator_welcome_left and mediator_welcome_user_right observed display_options, and updated their own internal flags to match the display options flags. This is rather clever, but also convoluted. The checkbox GUI handlers, instead of updating a global display options "view" model (and the resulting observer notifications happening) could instead take the simpler approach of modifying the flags in each mediator directly - which is what I implemented later. This then also allows the removal of the uppercase_all flag, since we no longer have a murky global view model, but instead a specific view model in each mediator (see dicussion of this topic of dropping the 3rd flag, in esper_extras.py)
